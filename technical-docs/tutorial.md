# Setup Tutorial — Database Layer

This tutorial reproduces the full database setup from scratch: PostgreSQL via Docker, SQLAlchemy ORM models, Alembic migrations, and loading COMPADRE data.

---

## Prerequisites

- Python 3.11+
- Docker Desktop running
- Git (to clone the repo)

---

## Step 1 — Clone the repository

```bash
git clone https://github.com/orvizz/population-growth-simulator.git
cd population-growth-simulator
```

---

## Step 2 — Create the environment file

Create a `.env` file at the project root (it is gitignored — never commit it):

```bash
# .env
POSTGRES_USER=postgres
POSTGRES_PASSWORD=your_password_here
POSTGRES_DB=matrix_db
POSTGRES_HOST=localhost
POSTGRES_PORT=5435
```

Port `5435` is used instead of the default `5432` to avoid conflicts with any locally installed PostgreSQL instance.

---

## Step 3 — Start PostgreSQL with Docker

```bash
docker compose up -d
```

This pulls `postgres:16-alpine` and starts a container named `tfg_postgres`. Data is persisted in a named Docker volume (`postgres_data`), so it survives container restarts.

Verify the container is healthy:

```bash
docker compose ps
# or
docker compose exec db pg_isready -U postgres -d matrix_db
```

---

## Step 4 — Install Python dependencies

```bash
pip install -r requirements.txt
```

Key packages installed:

| Package | Purpose |
|---|---|
| `sqlalchemy>=2.0` | ORM and DB abstraction |
| `alembic>=1.13` | Schema migrations |
| `psycopg2-binary` | PostgreSQL driver |
| `python-dotenv` | Load `.env` at runtime |

> **Note:** On some systems the `alembic` CLI is not added to `PATH`. Use `python -m alembic` instead throughout this tutorial.

---

## Step 5 — Apply the database migration

```bash
python -m alembic upgrade head
```

This runs all pending migrations in `alembic/versions/` in order. The initial migration (`0001_initial_schema`) creates three tables: `users`, `population_matrices`, and `simulation_runs`.

Verify:

```bash
docker compose exec db psql -U postgres -d matrix_db -c "\dt"
```

Expected output:

```
                List of relations
 Schema |        Name         | Type  |  Owner
--------+---------------------+-------+----------
 public | alembic_version     | table | postgres
 public | population_matrices | table | postgres
 public | simulation_runs     | table | postgres
 public | users               | table | postgres
```

---

## Step 6 — Seed COMPADRE data

The COMPADRE plant matrix database is stored in `growt-simulator-app/data/`. Pre-processed files (`metadata.parquet` and `matrices.json`) must exist — they are generated by `growt-simulator-app/preproccesor.py` and are not tracked in git due to size.

**If pre-processed files already exist** (fast path, ~10 seconds):

```bash
python -m db.seed_compadre
```

**If only the raw `.RData` file exists** (slow path, parses R data directly):

```bash
python -m db.seed_compadre --rdata
```

Expected output:

```
Loading from pre-processed files...
Seeding 9146 records into population_matrices...
  500/9146  1000/9146  ...  9000/9146
Done. Inserted: 9146, skipped (no matrix data): 0
```

Verify the data:

```bash
docker compose exec db psql -U postgres -d matrix_db -c \
  "SELECT COUNT(*), COUNT(DISTINCT species_accepted) FROM population_matrices;"
```

---

## Step 7 — (Optional) Generate pre-processed files from scratch

If neither the `.parquet` nor the `.json` exist, run the preprocessor first:

```bash
cd growt-simulator-app
python preproccesor.py
cd ..
```

This reads `COMPADRE_v.6.25.8.0.RData` and writes:
- `data/processed/metadata.parquet`
- `data/processed/matrices.json`

Then run the seeder as in Step 6.

## Daily workflow

```bash
# Start the DB (if not already running)
docker compose up -d

# Apply any new migrations after a git pull
python -m alembic upgrade head
```

## Useful commands reference

### Docker

```bash
docker compose up -d          # Start in background
docker compose down           # Stop (data preserved)
docker compose down -v        # Stop AND delete all data
docker compose logs db        # View PostgreSQL logs
docker compose exec db psql -U postgres -d matrix_db   # Open psql shell
```

### Alembic

```bash
python -m alembic upgrade head              # Apply all migrations
python -m alembic downgrade -1              # Roll back last migration
python -m alembic downgrade base            # Roll back everything
python -m alembic current                   # Show current revision in DB
python -m alembic history --verbose         # Show all migrations
python -m alembic revision --autogenerate -m "description"   # Generate new migration
```

### Adding a new migration (after editing `db/models.py`)

```bash
python -m alembic revision --autogenerate -m "add_xyz_column"
# Review the generated file in alembic/versions/
python -m alembic upgrade head
```

> Always manually review autogenerated migrations. Alembic can miss JSONB type changes, index modifications, and server defaults.

## Troubleshooting

| Problem | Cause | Fix |
|---|---|---|
| `alembic: command not found` | Alembic not on PATH | Use `python -m alembic` |
| `connection refused` | Container not running | `docker compose up -d` |
| `invalid input syntax for type json` | NaN values in matrix data | Already handled by `_sanitize_matrix()` in the seeder |
| `FATAL: password authentication failed` | Wrong credentials in `.env` | Check `.env` matches `docker-compose.yml` |
| Migration fails mid-run | DB in partial state | `python -m alembic downgrade -1` then fix and re-run |
