# Template Manual — TFG Plantilla

Quick reference for every function and convention in this Typst template.
All functions are defined in `template.typ` and imported where needed.

---

## Project File Structure

```
main.typ              ← orchestrator: includes everything, sets PDF metadata
template.typ          ← ALL styles and component functions live here
cover.typ             ← cover page (exactly 1 page, use v(1fr) for spacing)
backcover.typ         ← optional back cover
metadata.typ          ← tfg-title, tfg-author, and other reused variables
sources.bib           ← all bibliography entries

chapters/
  NN_name/
    index.typ         ← chapter content; manages its own sections
    tables.typ        ← all tables for this chapter (linked from index.typ)

resources/
  logos/
  diagrams/
  <chapter-name>/     ← images used in a specific chapter
```

---

## Adding a Chapter

1. Create `chapters/NN_name/index.typ`
2. In `main.typ`, add:

```typst
#pagebreak(weak: true)
#include "chapters/NN_name/index.typ"
```

The `weak: true` prevents a double blank page if the previous chapter already ended with a hard break.

## Heading Levels

```typst
= Chapter Title          // level 1 — large, right-aligned, bold small-caps
== Section               // level 2 — bold small-caps + rule below
=== Subsection           // level 3 — bold small-caps
==== Subsubsection       // level 4 — bold
```

Rules:
- Never skip levels (no jumping from `=` to `===`).
- Max recommended TOC depth: 3 levels.
- Label headings you'll reference: `= Introduction <sec:intro>`

Brown italic text that must be removed before the final submission.

```typst
#import "/template.typ": guia

#guia[Explain what goes here. This text won't appear in the final document.]
```

## Requirements — `req-group` + `req`

```typst
#import "/template.typ": req, req-group

// Set the prefix and reset the counter before a group of requirements.
#req-group("SR-F")

// indent: 0 → SR-F-01   indent: 1 → SR-F-01.1   indent: 2 → SR-F-01.1.1
#req("Authentication", [
  The system shall provide mechanisms for users to register and log in.
])

#req("User Login", [
  The system shall allow registered users to authenticate.
], indent: 1)

#req("Login Credentials", [
  The system shall accept email + password as valid credentials.
], indent: 2)
```

Cross-reference anywhere in the document:

```typst
See @req:sr-f-01 for the authentication requirement.
```

## User Stories — `user-story` + `story-map`

### Individual story card

```typst
#import "/template.typ": user-story

#user-story(
  role:     [registered user],
  want:     [log in with email and password],
  benefit:  [access my personalized content],
  priority: "Must",    // "Must" | "Should" | "Could" | "Won't"
  points:   [5],
  criteria: (
    [Valid credentials grant access to the dashboard.],
    [Invalid credentials show a generic error message.],
    [Session persists across page refreshes.],
  ),
)
```

Priority badge colours: Must = red · Should = orange · Could = green · Won't = grey.

Cross-reference anywhere:

```typst
As described in @us:01, users must be able to log in.
```

IDs auto-number in include order (US-01, US-02, …). Each story goes in its own file under `chapters/04_system_req/stories/` and is registered in `stories/_index.typ`.

### Story map grid

Wrap in `#figure` so it appears in the List of Tables:

```typst
#import "/template.typ": story-map

#figure(
  story-map((
    (epic: "Authentication", stories: ("Login", "Register", "Logout")),
    (epic: "Content",        stories: ("Create Post", "Edit Post", "Delete Post")),
  )),
  caption: [User Story Map],
) <tab:story-map>
```

## Figures & Tables

Every image, table, or diagram **must** be wrapped in `#figure`.

### Image (caption below)

```typst
#figure(
  image("/resources/diagrams/architecture.png", width: 85%),
  caption: [System architecture overview.],
) <fig:architecture>
```

### Table (caption above — template applies this automatically)

```typst
#figure(
  table(
    columns: (auto, 1fr, auto),
    [*ID*], [*Description*], [*Priority*],
    [US-01], [User login], [Must],
  ),
  caption: [Functional requirements summary.],
) <tab:req-summary>
```

## Cross-Reference Labels

Place a `<label>` immediately after the element in markup mode. Use `@label` in prose — Typst renders it as a clickable link in the PDF.

| Prefix | Use for |
|--------|---------|
| `fig:` | Images and diagrams |
| `tab:` | Tables |
| `sec:` | Section headings |
| `eq:`  | Equations |
| `lst:` | Code listings |
| `req:` | Requirements (auto-generated by `req()`) |
| `us:`  | User stories (auto-generated by `user-story()`) |

```typst
= Introduction <sec:intro>

// Inside a let binding, switch to markup mode first:
#let arch-fig = [
  #figure(image("..."), caption: [Architecture.]) <fig:arch>
]

// In prose:
As shown in @fig:arch, the system is divided into three layers.
See @sec:intro for background.
```

## OBS Node (Breakdown Structure Diagrams)

Used to build tree-style diagrams inside a grid layout:

```typst
#import "/template.typ": obs-node

#grid(
  columns: (1fr, 1fr),
  gutter: 8pt,
  obs-node(fill: rgb(200, 220, 255))[Frontend],
  obs-node(fill: rgb(200, 255, 220))[Backend],
)
```

## Cover Page Rules

The cover (`cover.typ`) must always fit exactly one page.

```typst
// Use v(1fr) to distribute space — never fixed mm values between elements.
#v(1fr)
// ... title block ...
#v(1fr)
// ... footer info ...
```

## Typography Micro-rules

| Rule | How |
|------|-----|
| Non-breaking space | `~` — e.g. `Table~3`, `Figure~@fig:arch` |
| External links | `#link("https://...")[descriptive text]` — never bare URLs |
| Inline code | `` `code` `` or `#raw("code")` — never bold/italic |
| Code block | ` ```language … ``` ` |
| Soft page break | `#pagebreak(weak: true)` before each `#include` in `main.typ` |
| No orphan headings | `#block(breakable: false)[== Heading \ First sentence…]` |
| Heading depth | Never skip levels; max 3 levels in TOC |

## Bibliography

Add entries to `sources.bib` (BibTeX format). Cite in text with `@key`:

```typst
The algorithm was proposed in @knuth1984.
```

Style and title are set once in `main.typ`:

```typst
#bibliography("sources.bib", style: "ieee", title: "Bibliography")
```

## PDF Metadata (set once in `main.typ`)

```typst
#import "metadata.typ": tfg-title, tfg-author
#set document(title: tfg-title, author: tfg-author, date: auto)
```
